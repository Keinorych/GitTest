Всё о GIT
Базовые команды:
git init – инициализация проекта
git add (имя файла или .) – добавить файл в очередь
git commit -m “любой текст” – фиксирует изменения  (-m означает сообщение)
git branch – используется для управления ветками репозитория
git branch -M (название) – переименовать локальную ветку
git remote add (название) (ссылка) – добавляет новое подключение к удалённому репозиторию с указанным именем и URL
git push – для отправки изменений в удалённый репозиторий
git push -u (название) (имя ветки) – для связи между локальной и удалённой веткой
git clone (ссылка) - клонирование репозиторий
git pull - получить все изменённые файлы
git status - узнать информацию об измененных файлах
[alias] в файле git.config можно добавить сокращения для команд git
git checkout -- (название файла или .) - отмена изменений
git reset (. или название файла) - отмена изменения файла из changes
git reset --hard HEAD^1 - жесткий сброс на 1 ветку (число можно менять) или git reset --soft HEAD^1 - мягкий сброс на 1 ветку
git checkout (название ветви) - смена ветки
git branch (название ветки) - создание новой ветки
